# 管理者マニュアル

## 1. このマニュアルの目的

このマニュアルは、会計システムの管理者が日常運用を迷わず行えるようにするための手順書です。  
対象は、支出申請の確認、精算状況の確認、問い合わせ対応、検索、状態更新を行う担当者です。

このシステムは Google Apps Script（GAS）で動作しており、管理者画面は「会計管理ダッシュボード」として提供されます。

---

## 2. 管理者ができること

管理者画面では主に次の作業ができます。

- 支出申請、通常精算、事後請求の一覧確認
- ID、日付、団体名、タイプ、金額、内容による検索
- 日付 / ID の並び替え
- 支出レコードのステータス変更
- 問い合わせ一覧の確認
- 問い合わせステータスの変更

管理者画面は「支出・精算管理」タブと「お問い合わせ管理」タブに分かれています。

---

## 3. ログインから管理画面を開くまで

### 3.1 権限の分岐

このシステムでは、ログインしたメールアドレスが以下のどちらに登録されているかで表示が変わります。

- 管理者マスタ `M_Admin`
- 利用者マスタ `M_Users`

管理者と利用者の両方に登録されている場合は、最初に権限選択画面が表示されます。

### 3.2 管理者画面を開く方法

1. システムの URL を開く
2. 権限選択画面が出た場合は `管理者ダッシュボードへ` を押す
3. 管理者専用で登録されている場合は自動で管理画面が開く

---

## 4. 管理画面の全体構成

### 4.1 画面上部

画面上部には次の要素があります。

- `会計管理ダッシュボード` のタイトル
- `申請画面へ切替` リンク
- `↻ 更新` ボタン

`↻ 更新` は、現在の画面状態を最新データで再取得したいときに使います。

### 4.2 タブ構成

- `支出・精算管理`
- `お問い合わせ管理`

最初に開くのは `支出・精算管理` タブです。  
`お問い合わせ管理` は最初から全件取得せず、タブを開いたタイミングで読み込みます。

---

## 5. 支出・精算管理タブの使い方

### 5.1 初期表示

初期表示では、**フィルターなしの全件表示** になります。  
団体名候補とタイプ候補も初回表示時に読み込まれているため、検索欄が `読み込み中...` のまま止まることは通常ありません。

### 5.2 一覧の列

支出・精算一覧には以下の列があります。

- `ID`
- `日付`
- `団体名`
- `タイプ`
- `金額`
- `内容`
- `領収書`
- `ステータス変更`

### 5.3 タイプの意味

現在の会計タイプは次の 3 種類です。

- `通常請求`
  - 旧名称の `事前` に相当
  - 領収書は不要
  - その後 `通常精算` で処理する前提の申請
- `事後請求`
  - 旧名称の `事後` に相当
  - 領収書 PDF が必須
- `通常精算`
  - 旧名称の `精算` に相当
  - 領収書 PDF が必須

### 5.4 領収書列

`領収書` 列に `確認` が表示されている場合、そのリンクから添付 PDF を開けます。  
`-` の場合は添付がないレコードです。

## 6. 検索機能の使い方

検索欄は複数条件を組み合わせて使えます。  
条件を入力したら `検索` ボタンを押してください。  
条件をすべて外す場合は `条件クリア` を押します。

### 6.1 ID検索

- `ID検索` に申請 ID の一部を入力すると部分一致で絞り込みます
- 例: `EXP-20260318`

### 6.2 日付検索

日付検索は **1つのカレンダーで開始日と終了日を選ぶ方式** です。

使い方:

1. `期間を選択してください` を押す
2. 1回目のクリックで開始日を選ぶ
3. 2回目のクリックで終了日を選ぶ
4. 必要なら左右の矢印で月を移動する
5. 解除する場合は `クリア` を押す

補足:

- 1回目だけ選ぶと、開始日と終了日は同じ日として保持されます
- 2回目を押すと期間検索になります
- 旅行サイトの期間選択に近い挙動です

### 6.3 団体名検索

- 団体名候補がチェックボックスで並びます
- 複数団体を同時に選択できます
- 何も選ばなければ全団体対象です

### 6.4 タイプ検索

- `通常請求`
- `事後請求`
- `通常精算`

複数タイプを同時に選択できます。

### 6.5 金額検索

`金額検索` では次を選べます。

- `指定なし`
- `一致`
- `以上`
- `以下`
- `範囲`

`範囲` を選ぶと最小金額と最大金額の両方を入力できます。

### 6.6 内容検索

- 内容欄に含まれるキーワードで部分一致検索します
- 例: `交通費`、`備品`

### 6.7 並び替え

並び替え対象:

- `日付`
- `ID`

並び順:

- `降順`
- `昇順`

---

## 7. ステータス変更の使い方

### 7.1 支出レコードのステータス

支出・精算レコードのステータス候補は次のとおりです。

- `申請中`
- `承認済`
- `未精算`
- `精算完了`
- `却下`

### 7.2 問い合わせのステータス

問い合わせレコードのステータス候補は次のとおりです。

- `未対応`
- `対応中`
- `完了`

### 7.3 ステータス変更手順

1. 対象レコードの右端のプルダウンを開く
2. 新しいステータスを選ぶ
3. システムが自動で保存する
4. 保存に成功すると画面上の一覧も更新される

補足:

- 保存に失敗した場合は元の値に戻ります
- 状態更新後に不安があれば `↻ 更新` を押して再読込してください

---

## 8. お問い合わせ管理タブの使い方

### 8.1 読み込みの仕様

`お問い合わせ管理` タブは、タブを開いた時に初めて読み込みます。  
初期状態では「お問い合わせタブを開くと読み込みます。」という表示です。

### 8.2 列の内容

- `ID`
- `日付`
- `団体名`
- `送信者`
- `件名`
- `本文`
- `ステータス変更`

### 8.3 対応手順の例

1. `お問い合わせ管理` タブを開く
2. 新規の `未対応` を確認する
3. 作業着手時に `対応中` に変更する
4. 完了したら `完了` に変更する

---

## 9. 管理者のおすすめ運用フロー

### 9.1 毎日の確認手順

1. 管理画面を開く
2. `支出・精算管理` タブで新着 ID を確認する
3. `日付検索` を今日または直近期間に絞る
4. 必要に応じて `団体名` や `タイプ` で絞り込む
5. 添付 PDF を確認する
6. ステータスを更新する
7. `お問い合わせ管理` タブを開いて未対応を処理する

### 9.2 月次確認の例

1. `日付検索` を月初から月末に設定する
2. `タイプ検索` で `通常請求` または `通常精算` を選ぶ
3. `団体名検索` で対象団体を絞る
4. `金額検索` を `以上` や `範囲` にして高額申請を確認する

---

## 10. ステータスの見方と注意点

### 10.1 通常請求の初期状態

現行実装では、`通常請求` は送信時点で `未精算` として登録されます。  
そのため、管理者画面で通常請求を探す場合は `未精算` を中心に確認してください。

### 10.2 事後請求 / 通常精算の初期状態

- `事後請求` の新規登録時: `申請中`
- `通常精算` の新規登録時: `申請中`

### 10.3 重複申請制限

同一団体には次の制限があります。

- `通常請求` の未完了レコードがあると新しい `通常請求` はできない
- `事後請求` の `申請中` があると新しい `事後請求` はできない

この制限は画面側だけでなくサーバー側でも適用されています。

## 11. トラブルシューティング

### 11.1 一覧が古いように見える

対応:

- `↻ 更新` を押す
- それでも変わらない場合はブラウザを再読込する

### 11.2 検索結果が出ない

確認ポイント:

- 日付範囲が狭すぎないか
- 団体名チェックが残っていないか
- タイプチェックが残っていないか
- 金額検索が `範囲` のままになっていないか

対応:

- `条件クリア` を押して全件に戻す

### 11.3 PDF が開けない

確認ポイント:

- リンク先が失効していないか
- ブラウザのポップアップ制限がかかっていないか

### 11.4 ステータス変更が反映されない

対応:

1. 変更後に元の値へ戻っていないか確認する
2. アラートが出ていないか確認する
3. `↻ 更新` で再取得する
4. Apps Script 実行ログを確認する

### 11.5 管理画面が重い

このシステムは軽量化済みですが、件数が増えると重く感じる場合があります。  
まずは次を試してください。

- 日付で絞る
- 団体名で絞る
- 内容検索を併用する

また、ブラウザの開発者ツール Console と Apps Script 実行ログには詳細ログが出ます。

---

## 12. デバッグログの見方

### 12.1 ブラウザ側

管理画面の JavaScript は `[KaikeiDebug]` プレフィックスで Console にログを出します。  
検索、読み込み、タブ切替、ステータス更新などのタイミングを確認できます。

### 12.2 サーバー側

GAS 実行ログでは以下を確認できます。

- `doGet`
- 管理画面 bootstrap の処理時間
- 問い合わせ読み込み
- シート読み込み件数
- キャッシュ hit / miss

---

## 13. 管理者向けの保守メモ

運用で困った時は次の順で確認すると把握しやすいです。

1. `docs/usage/tech-manual.md` を読む
2. `docs/ailog/2026-03-18-codex.md` で改修履歴を確認する
3. `src/admin.html` と `src/admin_script.html` を確認する
4. 必要なら `test/admin-dashboard-service.test.ts` と `test/admin-script-architecture.test.ts` を確認する

---

## 14. 最後に

管理画面で最も大事なのは、**検索条件を把握した上で、ステータスを落ち着いて更新すること** です。  
このシステムは「全件表示 → 条件で絞る → PDF 確認 → ステータス更新」という流れで使うと最も安定します。
